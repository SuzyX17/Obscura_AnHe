# 暗盒 (Obscura)

摄影师的灵感素材库。移动端优先的单文件 Web 应用,为摄影师 Sylvia 与她的内测朋友服务。

## 架构约定(改动前必读)

- **单文件**:整个应用就是 `index.html`,HTML/CSS/JS 全部内联,零构建、零依赖。不引入打包工具,不引入框架,不从 CDN 加载运行时依赖(导出等核心功能必须离线可用,ZIP 构建器是手写的,保持这样)。
- **数据全在用户端**:图片存 IndexedDB(库名 `anhe`,stores: `photos`/`colls`),词库与类别存 localStorage(`anhe_vocab`/`anhe_kinds`)。没有服务器,没有账号。这是产品承诺,不是技术欠账。
- **版本号**:每次改动必须递增头部 wordmark 里的版本号(如 `OBSCURA · v0.11` → `v0.12`)。这是用户确认"跑的是哪一版"的唯一手段,漏掉会导致排查灾难。
- **数据迁移**:改字段名或维度名时,在 `migrateTags`/`migrateAll` 里加映射,老用户数据要静默升级,永不丢失。已有先例可参考(如 `用途→参考→吸引点` 的链式改名)。
- **不用原生弹窗**:`confirm()`/`alert()` 在部分 WebView 沙盒里被静默禁用,一律用应用内的 `appConfirm()` 和 `toast()`。
- **图片解码**:走 `decodeImage()` 的多层降级(createImageBitmap → FileReader+dataURL),不要退回 blob URL + Image 的单一路径。

## 设计原则(来自与 Sylvia 的迭代共识)

- **"为什么"高于"是什么"**:"被什么打动"(note + 吸引点标签)是一等公民,金色边线区块;归类维度(题材/天时/光线等)是二等的,可全部留空。不要颠倒这个层级。
- **无提示语**:界面不放解释性 placeholder 和说明文案(仅有的例外:来源输入框的"作者 / 平台 / 链接"、备份面板的用途说明)。交互语言统一:chip 即入口、＋ 即内联新建、× 即删除、编辑态可拖动排序。
- **非评判**:空缺是诚实的数据,不是待办。不加催促、不加"还没写"类占位提醒。
- **词库是用户的**:所有预设标签、灵感集类别都可增删、可拖动排序、可恢复默认。不写死。

## 测试

改动后至少跑一次 happy-dom + fake-indexeddb 的运行时测试(参考仓库外的 test-runtime.mjs 模式,或让 Claude Code 重建一份):初始化 → 导入 → 显影卡 → 删除链路(经 appConfirm)→ ZIP 构建后用 `python3 -m zipfile` 验证完整性。语法检查:提取 <script> 后 `node --check`。

## 部署

推到 main 即上线(Netlify 自动部署)。纯静态,无构建命令,发布目录为根目录。
